# Skill 迭代升级说明

> 以下为 AI 信息筛选器 Skill 的后续迭代方向，每个方向独立可落地，按优先级排序。

---

## 迭代方向 1：真实 LLM 接入 + 语义理解升级

**当前状态**：使用基于关键词匹配的模拟评分，只能做字面匹配。

**升级目标**：接入 OpenAI / Claude / 本地 LLM API，实现真正的全文语义理解。

**具体改进点**：
- 将 `simulate_ai_filter()` 替换为真实的 LLM API 调用
- 利用 LLM 的上下文理解能力，识别"文章没提关键词但确实相关"的场景（如一篇讲 MCP 协议的文章，没出现 RAG 但和 Agent 开发直接相关）
- 让 LLM 识别标题党：标题夸张但正文空洞的文章自动降分
- 支持多模型切换（gpt-4o-mini 日常用，claude-3-haiku 备选，本地 ollama 免费方案）

**预期效果**：评分准确率从关键词匹配的 ~60% 提升到语义理解的 ~90%。

---

## 迭代方向 2：多模态内容支持

**当前状态**：只处理 RSS 文本文章。

**升级目标**：支持视频（B站/YouTube）、播客、微信公众号、PDF 论文等多种内容形态。

**具体改进点**：
- **视频**：抓取视频标题+简介+字幕/自动生成的字幕文本，AI 判断是否值得观看
- **播客**：通过 Whisper 等语音转文字工具将播客转为文本后分析
- **微信公众号**：通过 RSSHub 等工具将公众号转为 RSS 源
- **PDF 论文**：通过 arXiv API 抓取最新论文，AI 读摘要判断是否值得精读
- **统一评分体系**：不同类型内容在同一个 0-10 分尺度上可比

**预期效果**：覆盖用户 80% 以上的信息消费场景，不只是文字阅读。

---

## 迭代方向 3：个性化学习 + 反馈闭环

**当前状态**：兴趣画像是一次性手动配置的，不会自动进化。

**升级目标**：通过用户的阅读行为反馈（点击/忽略/收藏/分享），AI 自动调整兴趣画像。

**具体改进点**：
- **行为埋点**：记录用户对每篇推送的实际行为（点击阅读 / 收藏 / 标记不感兴趣 / 忽略）
- **反馈学习**：AI 根据反馈调整评分权重。被忽略的高分文章所属主题降权，主动收藏的低分文章所属主题升权
- **兴趣漂移检测**：当用户连续两周不点击某类主题时，AI 主动询问"你最近对 XX 还感兴趣吗？"
- **发现推荐**：偶尔推送 1-2 篇评分不高但可能激发新兴趣的"探索性"文章（类似 20% 探索 80% 利用的 Bandit 策略）

**预期效果**：三个月后推送内容与用户实际兴趣的匹配度持续提升，减少手动调参。

---

## 迭代方向 4：团队/社群共享筛选

**当前状态**：单用户使用，无法与他人共享筛选结果或协作订阅。

**升级目标**：支持多人共享订阅源和筛选结果，团队内部信息对齐。

**具体改进点**：
- **共享订阅源配置**：团队/学习小组共享一套订阅源配置
- **协作筛选**：同一篇文章多个成员可以标注"有价值"/"无价值"，系统综合多人判断
- **团队日报**：生成团队版日报，标注"3 人认为有价值"
- **知识库沉淀**：高价值文章自动存入团队知识库，支持全文检索和 AI 问答

**预期效果**：适合学习小组、技术团队等场景，减少团队内部"你看到那篇文章了吗"的信息差。

---

## 迭代方向 5：定时任务 + 多渠道推送

**当前状态**：需要手动运行脚本。

**升级目标**：全自动运行，多渠道推送筛选结果。

**具体改进点**：
- **定时运行**：通过 cron / GitHub Actions / 飞书机器人定时触发
- **多渠道推送**：
  - 飞书/钉钉/企业微信机器人推送日报摘要
  - 邮件发送完整日报（适合深度阅读）
  - RSS 输出（筛选后的结果本身又是一个高质量 RSS 源）
- **推送策略**：重要文章即时推送（发现即通知），可读文章每日汇总推送
- **免打扰时段**：用户可设置不想接收推送的时间段

**预期效果**：从"手动查"变成"自动收"，实现真正的信息助理体验。

---

## 迭代优先级总结

| 优先级 | 方向 | 核心价值 | 实现难度 |
|--------|------|----------|----------|
| P0 | 真实 LLM 接入 | 从关键词匹配到语义理解，质的飞跃 | 中 |
| P1 | 定时任务+多渠道推送 | 从工具变成服务，用户粘性关键 | 低 |
| P2 | 个性化学习反馈闭环 | 越用越懂你，长期竞争力 | 中 |
| P3 | 多模态内容支持 | 覆盖更多场景，扩大适用面 | 高 |
| P4 | 团队共享筛选 | 新场景拓展 | 高 |
